## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, dense inline metadata |
| sm | 8px | Compact form spacing, chip gaps, table-cell inner spacing |
| md | 16px | Default card padding, form field spacing, stacked mobile rhythm |
| lg | 24px | Section padding, card gutters, drawer interior spacing |
| xl | 32px | Page section gaps, KPI row spacing |
| 2xl | 48px | Major layout breaks, dashboard section separation |
| 3xl | 64px | Page-level top/bottom spacing on desktop |

Exceptions: 44px minimum tap targets for icon-only and mobile actions; 56px topbar height; 240px desktop sidebar width; 500-row stream cap in `DataTable` is a performance constraint, not a spacing token.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 600 | 1.4 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.2 |

Typography contract: only two weights are allowed in v1.0 admin surfaces, `400` and `600`. Use `16px/400` for all table, form, drawer, and inspector body copy. Use `14px/600` for labels, chip text, badge text, and KPI captions. Use `20px/600` for page headings and drawer titles. Use `28px/600` for KPI values only.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Paper `#FAFBFC` in light mode / Ink `#111418` in dark mode (`FND-07`, `D7-08`) | App background and primary canvas |
| Secondary (30%) | Fog `#E9EEF2` in light mode / Slate `#24303B` in dark mode (`FND-07`, `D7-08`) | Cards, sidebar, topbar, drawers, dividers, sunken surfaces |
| Accent (10%) | Cobalt `#5D79F6` by default, runtime-overridable via `brand.accent_hex` (`D7-08`) | Primary emphasis only |
| Destructive | Ink `#111418` text on danger surfaces tinted from the accent system plus explicit warning state via Amber `#C8923B`; destructive confirmations must never reuse Moss/Cobalt (`ADMIN-21`, `D7-05`) |

Accent reserved for: primary buttons, active navigation item, selected tabs, selected filter chips, focus rings, inline links, command-palette active row, and poll-banner "N new rows" action. Do not use accent for neutral table chrome, secondary buttons, badges, or large background fills.

Semantic state colors are fixed: Moss `#5E9E84` for success/active, Cobalt `#5D79F6` for interactive/info, Amber `#C8923B` for warning, grace, dunning, and dev-only surfaces. Raw palette tokens are escape hatches; implementation should prefer semantic aliases from `D7-08`.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Review webhook failures |
| Empty state heading | No billing records yet |
| Empty state body | Matching records will appear here as customers subscribe, invoices are issued, and webhooks arrive. Clear filters or come back after the next billing event. |
| Error state | We couldn’t load this billing data. Retry the page, narrow the filters, or inspect webhook health if the problem persists. |
| Destructive confirmation | Refund charge: Confirm the refund amount and fee impact before continuing. Cancel subscription: Choose end-of-period or immediate cancellation, then confirm. Void or mark invoice uncollectible: Confirm you want to stop collection for this invoice. Bulk requeue DLQ: Confirm queue, row count, concurrency, and estimated drain time before enqueuing. |

Copy contract notes: use direct operational verbs, never marketing language. Buttons should read `Replay`, `Refund`, `Pause`, `Resume`, `Void invoice`, `Mark uncollectible`, `Open PDF`, `Copy payload`, `Load new rows`. Poll banner copy is fixed: `{N} new rows — click to load` (`D7-04`). Dev routes must display `[NON-PROD]` in the Dev nav group (`D7-07`).

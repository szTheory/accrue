<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser runner -> Phoenix host app | Untrusted browser actions hit authenticated host and admin routes |
| seeded fixture file -> Playwright spec | Fixture data crosses from Elixir seeding script into Node test code |
| Playwright artifact output -> CI artifact upload | Reports, traces, screenshots, and logs may contain sensitive state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01-01 | Spoofing | host browser login/admin flow | mitigate | Reuse the real `/users/log-in` session flow and billing-admin routes; do not inject private auth shortcuts or bypass host session checks. |
| T-11-01-02 | Tampering | Playwright spec assertions | mitigate | Assert exact Phase 10 copy and routes from the shipped host/admin UI so CI fails on real UX regressions rather than a synthetic test fixture. |
| T-11-01-03 | Information Disclosure | traces/screenshots/report | mitigate | Keep the browser gate Fake-backed only, store only failure artifacts, and avoid introducing Stripe live keys, webhook secrets, or raw secret prints into the spec. |
| T-11-01-04 | Denial of Service | flaky LiveView/browser waits | mitigate | Require explicit LiveView connection checks and bounded Playwright timeouts instead of open-ended retries that could hang CI. |
| T-11-01-05 | False Negative | raw node smoke remains primary | mitigate | Replace the custom node smoke path with Playwright Test so the blocking browser surface has retained traces/screenshots and standard failure semantics. |
</threat_model>

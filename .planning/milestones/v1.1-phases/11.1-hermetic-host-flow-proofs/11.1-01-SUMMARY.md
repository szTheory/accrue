## Decisions Made
- Let the focused proof case own persisted-state hygiene rather than moving more reset logic into the wrapper. That preserves the Phase 11 release-gate script while fixing the direct-rerun contract at the proof boundary.
- Keep signed webhook verification and admin authorization on their real router and session paths. The phase changed only proof setup and replay fixture ids, not the security-sensitive assertions.
- Delete all `Accrue.Webhook.DispatchWorker` jobs in the proof cleanup. Browser replay leaves both discarded and available jobs for the same host webhook, so state-based cleanup was insufficient.

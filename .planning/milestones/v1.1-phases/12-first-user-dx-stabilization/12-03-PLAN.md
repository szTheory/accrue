<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| installer writes -> host-owned generated files | `mix accrue.install` writes into host code and config paths the user may later edit |
| installer sidecar artifacts -> developer workspace | Conflict outputs must stay outside compile/config paths and must not leak secrets |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-03-01 | Tampering | `accrue/lib/accrue/install/fingerprints.ex` | mitigate | Preserve stamped edited-file detection and forbid `--force` from overwriting stamped user-edited files per D-02 and D-03. |
| T-12-03-02 | Information Disclosure | `.accrue/conflicts/*` artifacts | mitigate | Write artifacts only under `.accrue/conflicts/templates/` and `.accrue/conflicts/patches/`, carry only `target:` and `reason:` headers, and reuse the installer's existing redaction for all printed output. |
| T-12-03-03 | Denial of Service | installer summary output | mitigate | Emit the exact categories `created`, `updated pristine`, `skipped user-edited`, `skipped exists`, `manual`, and `conflict artifact` so CI and reruns surface the real failure mode instead of a coarse count. |
</threat_model>

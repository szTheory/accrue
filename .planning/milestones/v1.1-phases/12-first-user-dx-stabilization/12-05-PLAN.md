<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host config/router -> Accrue boot and installer checks | Untrusted host setup determines whether webhooks, admin mounts, repo config, and auth are safe and runnable |
| incoming webhook request -> Accrue.Webhook.Plug | Signature headers and request bodies are attacker-controlled |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-05-01 | Information Disclosure | `accrue/lib/accrue/setup_diagnostic.ex` and `accrue/lib/accrue/webhook/plug.ex` | mitigate | Redact `sk_*`, `whsec_*`, and `SECRET`/`KEY` values everywhere diagnostics are formatted; log keys and classes only. |
| T-12-05-02 | Tampering | `accrue/lib/accrue/webhook/plug.ex` | mitigate | Keep bad signatures on a generic `400` and preserve raw-body/signature verification on the existing plug path; never teach or implement a bypass. |
| T-12-05-03 | Elevation of Privilege | `accrue/lib/accrue/auth/default.ex` and installer preflight | mitigate | Fail closed when the default/dev auth adapter is used in production and detect missing admin protection or missing admin mount wiring in `--check`. |
| T-12-05-04 | Security Misconfiguration | installer preflight | mitigate | Check webhook route placement, raw-body reader scope, browser-pipeline misuse, missing admin mount, and auth boundary issues with stable diagnostic codes. |
</threat_model>

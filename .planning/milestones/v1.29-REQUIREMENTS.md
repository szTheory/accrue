## v1.29 — Mailglass Integration

**Goal (shipped):** Integrate the mailglass packages (as path dependencies) to completely replace MJML and the raw Swoosh pipeline in accrue.

### Mailglass Foundation & Admin (MG)

- [x] **MG-01**: Replace `mjml_eex` and `phoenix_swoosh` with `{:mailglass, path: "../../mailglass"}` in `accrue` and `{:mailglass_admin, path: "../../mailglass", only: [:dev, :test]}` in `accrue_admin`.
  *Outcome (Phase 88):* Path deps wired; `:dev, :test` (not just `:dev`) was required because `accrue_admin/2` macro expansion imports `MailglassAdmin.Router` at compile time. `mjml_eex` / `phoenix_swoosh` removed from `accrue/mix.exs` in Phase 90.

- [x] **MG-02**: Mount the `mailglass_admin` LiveView UI in `accrue_admin`'s router to provide an instantaneous visual feedback loop, replacing the old `mix accrue.mail.preview` command.
  *Outcome (Phase 88):* Mounted as sibling scope OUTSIDE the `:accrue_admin` `live_session` (Phoenix forbids nested `live_session`). Route existence verified at compile time via ExUnit `__routes__/0` assertions — shift-left replaces human browser-verify.

- [x] **MG-03**: Update Accrue's installation instructions and/or `mix accrue.install` to apply the required Mailglass Postgres migrations (`mailglass_deliveries`, `mailglass_events`, `mailglass_suppressions`).
  *Outcome (Phase 88):* Migration docs added to install + email guides. Phase 90 added `accrue/priv/repo/migrations/20260426000000_create_mailglass_poc_tables.exs` so host repos boot cleanly.

### Pipeline & Idempotency (MG)

- [x] **MG-04**: Refactor `Accrue.Workers.Mailer` to build and dispatch via `Mailglass.deliver/1`, orchestrating dynamic assignment hydration and Multi-tenant PDF attachment logic before dispatch.
  *Outcome (Phase 89-01):* Worker seam delegates to `Mailglass.deliver/1`. Receipt PDF attachment preserved end-to-end.

- [x] **MG-05**: Implement database-level idempotency by passing an explicit `idempotency_key` to Mailglass messages, replacing the Oban `unique: [period: 60]` logic to prevent duplicate webhook dispatches.
  *Outcome (Phase 89-01):* Explicit `idempotency_key` per message; Oban `unique: [period: 60]` removed. Duplicate dispatches rejected at the database index level.

### Template Porting (MG)

- [x] **MG-06**: Port the Proof-of-Concept templates (`Accrue.Emails.Receipt` and `Accrue.Emails.PaymentFailed`) from MJML to `Mailglass.Mailable` using native Mailglass HEEx components.
  *Outcome (Phase 89-02):* Both templates ported. Parity tests assert adopter-visible content/structure rather than byte-for-byte HTML (D-07/D-08).

- [x] **MG-07**: Port the remaining 11 MJML templates in `priv/accrue/templates/emails/` to `Mailglass.Mailable` and completely remove the MJML compiler dependency.
  *Outcome (Phase 90):* All 13 transactional templates Mailglass-backed (`payment_succeeded` ported in Plan 03 to drop the last MJML holdout). `mjml_eex` and `phoenix_swoosh` removed from `accrue/mix.exs`. 26 legacy `.mjml.eex`/`.text.eex` files deleted. `mix accrue.mail.preview`, `Accrue.Emails.HtmlBridge`, and `Accrue.Workers.Mailer.template_for/1` removed. `mailglass_cleanup_test` guards regressions in CI.

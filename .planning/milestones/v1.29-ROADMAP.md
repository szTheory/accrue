# Roadmap: Mailglass Integration (v1.29)

**Goal:** Integrate the mailglass framework to replace MJML and `phoenix_swoosh` with HEEx-native components, event ledgers, and a dev-preview UI.

## Phases

- [ ] **Phase 88: Mailglass Foundation** - Add path dependencies, mount dev UI, and verify migrations
- [ ] **Phase 89: Proof of Concept Templates & Pipeline** - Connect the Oban worker, implement idempotency, and port the first two templates
- [ ] **Phase 90: Full Template Port & Cleanup** - Port all remaining MJML templates and remove legacy dependencies

## Phase Details

### Phase 88: Mailglass Foundation
**Goal**: Mailglass dependencies are present, migrations execute, and dev preview UI is available
**Depends on**: Phase 87
**Requirements**: MG-01, MG-02, MG-03
**Success Criteria**:
  1. Developer can run Mailglass migrations in the host application successfully.
  2. Developer can access the `/dev/mail` LiveView dashboard via `accrue_admin` locally.
  3. `mailglass` and `mailglass_admin` are correctly installed as path dependencies.
**Plans**: TBD

### Phase 89: Proof of Concept Templates & Pipeline
**Goal**: The Accrue Mailer worker successfully sends Mailglass emails with idempotency and PDF attachments
**Depends on**: Phase 88
**Requirements**: MG-04, MG-05, MG-06
**Success Criteria**:
  1. `Accrue.Workers.Mailer` delegates to `Mailglass.deliver/1` with correctly hydrated assigns.
  2. Receipt and PaymentFailed emails render visually matching their MJML counterparts using HEEx components.
  3. PDF invoices are successfully attached to the Mailglass message for Receipt emails.
  4. Duplicate dispatches are natively rejected using Mailglass's `idempotency_key` index.
**Plans**: TBD

### Phase 90: Full Template Port & Cleanup
**Goal**: All emails use Mailglass HEEx components and legacy MJML dependencies are removed
**Depends on**: Phase 89
**Requirements**: MG-07
**Success Criteria**:
  1. All 13 email templates render correctly via Mailglass HEEx components.
  2. `mjml_eex` and `phoenix_swoosh` are entirely removed from the application dependencies.
  3. Legacy `mix accrue.mail.preview` tooling is retired.
**Plans**: TBD

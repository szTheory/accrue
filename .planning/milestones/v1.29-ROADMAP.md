## Overview

Integrate the [Mailglass](https://github.com/szTheory/mailglass) framework as a path dependency to replace `mjml_eex` and `phoenix_swoosh` with HEEx-native components, an append-only event ledger, native database-level idempotency via `idempotency_key`, and a LiveView dev-preview dashboard mounted at `/dev/mail` in `accrue_admin`. Eliminates the Node-dependent MJML compile step and the bespoke Oban-uniqueness idempotency window. **No** **PROC-08** / **FIN-03** scope creep.

## Milestone Summary

**Decimal Phases:** None.

**Key Decisions:**
- **D-Mailglass-path-dep**: Mailglass is consumed via `path: "../../mailglass"` (sibling repo, two levels below `~/projects/`) — not Hex — for v1.29.
- **D-sibling-scope**: `mailglass_admin_routes("/dev/mail")` mounted as a sibling scope OUTSIDE the `:accrue_admin` `live_session` block; Phoenix forbids nested `live_session`.
- **D-shift-left UAT**: Phase 90 retires the human browser-verify checkpoint entirely. All 7 UAT truths automated via ExUnit (`mailglass_cleanup_test`, `fixtures_test`, `multipart_coverage_test`, `email_preview_live_test`).
- **D-explicit-idempotency**: Phase 89 replaces the Oban `unique: [period: 60]` idempotency window with explicit `Mailglass.deliver/1` `idempotency_key` (database-level rejection of duplicates).
- **D-no-shim**: Phase 90 deletes `mix accrue.mail.preview` and `Accrue.Emails.HtmlBridge` outright; no compatibility shim. Adopters move to the supported `/dev/email-preview` admin LiveView.
- **D-cleanup-guard-test**: `mailglass_cleanup_test` enforces (in CI) that `mjml_eex`, `phoenix_swoosh`, the CLI task, `template_for/1`, and the legacy `*.mjml.eex` / `*.text.eex` assets stay gone.
- **D-payment-succeeded-Mailglass**: The `payment_succeeded` compatibility alias is also Mailglass-backed and added to `Fixtures.all/0` so the broad sweep stays honest after the dependency purge.

**Issues Resolved:**
- Replaced the Node-dependent `mjml_eex` compile step in the email render path with a pure Elixir HEEx pipeline.
- Replaced the in-process Oban `unique: [period: 60]` idempotency window with a database-level `idempotency_key` index — duplicates rejected at insert, not after dispatch.
- Removed the long-standing `mix accrue.mail.preview` CLI task in favor of the supported `AccrueAdmin.Dev.EmailPreviewLive` UI.
- Eliminated `Accrue.Emails.HtmlBridge` and `Accrue.Workers.Mailer.template_for/1` — both existed only to support the retired CLI.

**Issues Deferred:** None for v1.29 scope.

**Technical Debt Incurred:**
- Mailglass is consumed as a path dep — adopters cannot install via Hex until Mailglass itself is published. Out of scope for v1.29; tracked for a future "Mailglass on Hex" milestone.
- `accrue/priv/repo/migrations/20260426000000_create_mailglass_poc_tables.exs` is a host-bootstrap migration that lives alongside the cleanup commit. Long-term home (host generator vs. install task) is a follow-up question.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| webhook payload → worker args | Stripe-derived identifiers and mail type flow into job args and idempotency derivation. |
| worker → Mailglass/PDF renderer | Mail content, attachment bytes, and message metadata cross into outbound delivery. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-89-01 | T | `Accrue.Workers.Mailer` | mitigate | Keep Oban args scalar-only, derive `idempotency_key` from the approved `accrue:v1:<type>:<charge_id>` shape only, and reject any missing/invalid charge id before building the Mailglass message. |
| T-89-02 | I | PDF attachment path / telemetry | mitigate | Attach only the binary returned by `Accrue.Billing.render_invoice_pdf/2`, fall back to the hosted invoice URL when PDF rendering fails, and keep telemetry / idempotency metadata free of recipient or body content. |
</threat_model>

# Milestone v1.6: Admin UI / UX polish

**Status:** SHIPPED 2026-04-20  
**Phases:** 25–29  
**Total plans:** 16 (3 + 4 + 3 + 3 + 3)

## Overview

Bring the companion admin (`accrue_admin`) to a **consistent, accessible, mobile-credible** bar: hierarchy and tokens match Phase 20/21 UI contracts where they apply, high-traffic copy is predictable, and CI catches regressions on narrow viewports and critical a11y rules—without introducing a new third-party component kit. Host Playwright proofs remain scoped to mounted admin / VERIFY-01 where already established.

## Milestone summary

**Key decisions:**

- Stay on LiveView + `ax-*` + existing CSS; no new third-party UI kits (Phase 20/21 contracts).
- VERIFY-01 / host Playwright remain the browser proof surface for mounted admin; expand `@mobile` / `chromium-mobile` for admin-heavy flows.
- Theme tokens default on touched surfaces; exceptions documented (e.g. `26-theme-exceptions.md` / registry patterns).

**Issues resolved:**

- Route, component, and UI-SPEC alignment baselined for money indexes, detail pages, webhooks, and step-up flows.
- Operator microcopy and stable literals for tests; a11y gates (focus, table captions, axe on mounted customers index); mobile README shell + Playwright mobile coverage beyond placeholder specs.

**Technical debt / follow-ups:**

- `audit-open` carry-forward at close: Phase 21 UAT tooling flag; two missing quick-task stubs (see `.planning/STATE.md` § Deferred Items).
- Dedicated B2C-shaped host LiveView teaching path, **PROC-08**, **FIN-03**, **ORG-04** remain explicitly out of scope for v1.6 (see requirements archive).

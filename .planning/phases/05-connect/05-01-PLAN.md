## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Stripe → Webhook plug | Untrusted payload + signature header; must verify via per-endpoint secret |
| Oban job args → DispatchWorker | Args crossing process boundary; endpoint/stripe_account must survive serialization |
| LatticeStripe client → Stripe HTTPS | Secret key + Stripe-Account header; platform leak vector if header misset |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01-01 | Spoofing | Webhook plug (Connect endpoint) | mitigate | Verify `:connect` payload against `:connect` secret, not platform (VALIDATION row 25/26). Runtime warning if both secrets byte-identical (Pitfall 5). |
| T-05-01-02 | Tampering | accrue_webhook_events.endpoint column | mitigate | Ecto.Enum constrains to `[:default, :connect]`; DB default prevents nil; DispatchWorker handles `nil` row gracefully via catch-all fallthrough to DefaultHandler. |
| T-05-01-03 | Information Disclosure | resolve_stripe_account/1 precedence chain | accept | Platform-only calls (nil resolver result) are the correct default; risk is platform ops setting `:default_stripe_account` globally by mistake. Documented in guides/connect.md (Plan 07). |
| T-05-01-04 | Elevation of Privilege | Oban middleware stripe_account propagation | mitigate | Middleware only READS stripe_account from trusted enqueue-time args (never from payload fields); Oban uses jsonb args signed by the DB, not external input. |

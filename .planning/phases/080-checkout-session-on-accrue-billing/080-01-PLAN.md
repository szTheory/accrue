# Plan 080-01 — BIL-06 `Accrue.Billing.create_checkout_session`

<threat_model>
  <summary>ASVS L1 billing facade. Primary risks: leaking bearer credentials (checkout URL, client_secret) via telemetry or logs, and processor error paths surfacing raw secrets.</summary>
  <threat id="T-80-01">Telemetry or span metadata exposes checkout redirect URL, client_secret, or serialized line items / raw attr blobs.</threat>
  <mitigation>Pass only <code>NimbleOptions.validate!</code> output into <code>span_billing</code> metadata path; extend <code>billing_metadata/4</code> with strict allowlist keys <code>checkout_mode</code>, <code>checkout_ui_mode</code>, <code>checkout_line_items_count</code> derived from validated opts only. Tests assert <code>inspect(metadata)</code> lacks <code>http</code>, <code>client_secret</code>, and representative line-item payload strings.</mitigation>
  <threat id="T-80-02">Facade accepts arbitrary keys and forwards unvalidated maps to <code>Checkout.Session</code>, bypassing outer validation.</threat>
  <mitigation>Define <code>@checkout_session_facade_attrs_schema</code> as <code>Accrue.Checkout.Session</code> <code>@create_schema</code> fields <strong>excluding</strong> <code>:customer</code>; use <code>NimbleOptions.validate!</code> on every call. ExUnit asserts <code>NimbleOptions.ValidationError</code> on unknown key.</mitigation>
  <threat id="T-80-03">Alias clash in <code>billing.ex</code> causes wrong Session module in specs or delegation.</threat>
  <mitigation>Use alias <code>Accrue.Checkout.Session as CheckoutSession</code> (do not reuse <code>Session</code> from BillingPortal). <code>@spec</code> references <code>CheckoutSession.t()</code>.</mitigation>
</threat_model>

## Objective

Ship **`Accrue.Billing.create_checkout_session/2`** and **`create_checkout_session!/2`** per **BIL-06** and **080-CONTEXT.md**: NimbleOptions-validated attrs (no **`:customer`** in attrs — first arg is **`%Accrue.Billing.Customer{}`**), delegate to **`CheckoutSession.create(Map.new([customer: customer | validated]))`**, **`span_billing(:checkout_session, :create, …)`** with allowlisted metadata, plus **Fake** + telemetry **ExUnit** in **`checkout_session_facade_test.exs`**.

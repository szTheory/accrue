<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser-seeded committed rows -> direct ExUnit proof setup | Persisted test-database state from `scripts/ci/accrue_host_uat.sh` can contaminate later direct proof reruns if cleanup is too narrow or too broad |
| signed webhook payload -> router/webhook ingest | Untrusted payload bytes and signature headers must still be verified through the real ingest path |
| admin Phoenix session -> mounted `/billing` replay action | Replay remains privileged host-admin behavior and must keep the real session token plus `billing_admin` authorization checks |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11.1-01 | Tampering | `examples/accrue_host/test/accrue_host_web/webhook_ingest_test.exs` | mitigate | Keep `LatticeStripe.Webhook.generate_test_signature/2`, router POSTs, 400 tamper assertion, and zero-row assertions for `WebhookEvent` and `Oban.Job`; do not stub signature verification. |
| T-11.1-02 | Elevation of Privilege | `examples/accrue_host/test/accrue_host_web/admin_webhook_replay_test.exs` | mitigate | Preserve `log_in_user(conn, admin_user)`, `billing_admin: true`, mounted `/billing/webhooks/:id` LiveView path, and `admin.webhook.replay.completed` audit assertions. |
| T-11.1-03 | Tampering | `examples/accrue_host/test/support/host_flow_proof_case.ex` cleanup helper | mitigate | Delete only proof-owned rows keyed by exact emails, event types, worker name, `price_basic`, and `sub_fake_%` / `sub_host_%` / `evt_host_%` prefixes; do not truncate tables or drop the database in proof setup. |
| T-11.1-04 | Repudiation | replay audit event path | mitigate | Keep the replay proof asserting `admin.webhook.replay.completed`, `actor_type == "admin"`, `actor_id == admin_user.id`, and `caused_by_webhook_event_id == webhook.id`. |
| T-11.1-05 | Denial of Service | canonical wrapper vs direct rerun contract | accept | The wrapper still performs full `ecto.drop/create/migrate` resets for release-gate orchestration; this plan intentionally leaves that heavier reset in the shell script and makes the proof files tolerate reruns on a migrated database. |
</threat_model>

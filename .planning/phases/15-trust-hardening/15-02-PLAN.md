<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| signed webhook request -> host router | Untrusted webhook input crosses into verification, persistence, and queueing under a measured smoke budget. |
| seeded browser session -> mounted `/billing` flows | Authenticated browser state crosses into admin dashboard, webhook detail, and replay views. |
| retained screenshots -> CI/repo artifacts | Canonical success artifacts can leak secrets or PII if the seeded content or retention policy drifts. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-06 | D | `examples/accrue_host/test/accrue_host_web/trust_smoke_test.exs` | mitigate | Measure only the signed request path and seeded admin route path, use conservative documented budgets, and keep the checks deterministic instead of benchmark-oriented. |
| T-15-07 | T | `examples/accrue_host/mix.exs` | mitigate | Fold trust smoke into the existing `mix verify` / `mix verify.full` contract so the canonical host gate cannot skip webhook/admin trust checks. |
| T-15-08 | I | Playwright screenshots and reports | mitigate | Keep heavy artifacts failure-only, keep success screenshots limited to the five UI-spec labels, and use seeded fixture data that contains no production secrets or customer PII. |
| T-15-09 | E | responsive replay/admin UI | mitigate | Add mobile and desktop assertions that replay controls, primary actions, and seeded state remain reachable without hover-only or clipped interactions. |
| T-15-10 | R | accessibility proof | mitigate | Run Axe only after the page reaches the seeded state being validated and fail on `critical` and `serious` issues so the accessibility signal maps to the actual trust path. |
</threat_model>

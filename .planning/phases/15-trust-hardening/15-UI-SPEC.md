# Phase 15 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | Phoenix LiveView function components plus custom `AccrueAdmin.Components`; host demo uses Tailwind CSS with daisyUI |
| Icon library | Heroicons |
| Font | `system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif` |

Phase 15 extends existing demo and admin surfaces only. Do not introduce a landing page, new marketing shell, or a second visual system for trust evidence.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline gaps, eyebrow spacing, compact icon separation |
| sm | 8px | Filter gaps, chip spacing, dense control spacing |
| md | 16px | Default control spacing, card internals, body rhythm |
| lg | 24px | Card padding, section gaps, form groups |
| xl | 32px | Page-section spacing on mobile, subscription page shell padding |
| 2xl | 48px | Major section breaks, desktop page rhythm |
| 3xl | 64px | Reserved for large page-level separation only |

Exceptions: Interactive targets must remain at least 44px tall or wide; icon-only and pill controls may use 44px while sticky topbar rows may stay at 56px. Mobile layouts must collapse data tables into cards instead of horizontal page overflow.

---

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 600 | 1.4 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.2 |

Use only weights `400` and `600` in this phase. Trust artifacts, admin KPI cards, replay details, and responsive screenshots should all reuse these four text roles.

---

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FAFBFC` | Page background, default surface, screenshot background in light theme |
| Secondary (30%) | `#FFFFFF` | Cards, sidebar, topbar, table shells, elevated admin surfaces |
| Accent (10%) | `#5D79F6` | Primary CTA, active nav state, tabs, focus ring, replay/status emphasis, retained success screenshot points of attention |
| Destructive | `#111418` | Destructive confirmation actions only |

Accent reserved for: `Start subscription`, current navigation item, active tabs, visible keyboard focus, replay-related operator actions, and compact badges that identify actionable trust state. Do not use accent as a general page wash or for noninteractive decorative fills.

Dark theme must continue to use the existing admin token swap: dominant surface `#0F1318`, secondary elevated surface `#171D24`, and the same accent `#5D79F6`.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Start subscription |
| Empty state heading | No billing activity yet |
| Empty state body | Billing records appear after a user starts a subscription or a webhook is processed. Start a subscription or review the webhook feed. |
| Error state | We couldn't complete that billing action. Check auth, migrations, webhook signing, or processor setup, then try again. |
| Destructive confirmation | Cancel subscription: Confirm cancellation before ending access. |

Additional copy rules for this phase:

- Trust evidence copy must stay literal and evidence-backed. Use labels such as `required deterministic gate`, `advisory`, `replay requested`, `verification passed`, and `no local events recorded yet`.
- Never use maturity claims such as `battle-tested`, `enterprise-grade`, or similar unsupported marketing language.
- Artifact names and screenshot labels must describe the verified state, not the implementation detail.

---

## Interaction Contract

1. Browser coverage must target the existing seeded flow only: subscription empty state, subscription started state, admin dashboard, webhook detail with replay affordance, and replay audit state.
2. Required responsive viewports are:
   - Desktop: `1280x900`
   - Mobile: Playwright device profile equivalent to `Pixel 5`
3. Accessibility remains release-blocking for `critical` and `serious` axe violations on each required state above.
4. Keyboard interaction must remain explicit:
   - Visible accent focus ring on all buttons, links, tabs, filter controls, and replay controls
   - Skip link remains reachable before admin shell content
   - No interaction may depend on hover alone
5. Admin shell behavior by breakpoint:
   - `<1024px`: sidebar hidden, topbar remains sticky, navigation available through compact controls, data tables render card list
   - `>=1024px`: sidebar visible, page content stays in two-column shell, tables may render full grid plus cards only where already implemented
6. Performance-trust UI must stay compact:
   - Show summary metrics and pass/fail states, not benchmark dashboards
   - Prefer one-line KPI or status copy over verbose explanations inside the browser surface
7. Replay and other operator actions must use direct verbs, immediate flash feedback, and auditable follow-up state. Do not add celebratory or promotional language.

## Artifact Contract

| Artifact | Contract |
|----------|----------|
| Success screenshots | Keep a compact set of PNG screenshots for the canonical states only: `first-run-billing-empty`, `first-run-subscription-started`, `admin-dashboard`, `admin-webhook-detail`, `admin-replay-audit` |
| Failure artifacts | Permit Playwright trace, screenshot, and HTML report retention on failure only |
| Screenshot content | Must avoid secrets, webhook signing values, tokens, production payloads, customer PII, or raw provider credentials |
| Visual assertions | Assert readable layout, visible primary action, reachable replay control, and no clipped or overlapping text in both required viewports |
| Review usage | Screenshots support review; behavioral assertions and axe checks remain the blocking source of truth |

## FRG-01 — Ranked friction inventory

### Priority bar (triage P0 vs CI)

Use **two axes** on every row (avoids “CI green so no pain” and “everything is P0”):

| Axis | Values | Meaning |
|------|--------|---------|
| **`ci_contract`** | `merge_blocking` · `advisory` · `n/a` | Relationship to **VERIFY-01**, **host-integration**, **`verify_package_docs`**, or other named checks. |
| **`integrator_impact`** | `blocks_golden_path` · `blocks_secondary_path` · `no_workaround` · `workaround_documented` | Effect on the documented first-hour / host path. |

**Triage P0** = (**`blocks_golden_path`** OR **`no_workaround`** for a documented integrator obligation) **and** impact touches **core adoption**, **correctness**, or **trust** for billing — **or** row documents a **`merge_blocking`** failure on `main`.

**CI-only red** without integrator story → fix immediately; may be **`merge_blocking`** without being a long-lived **FRG-03** row (close when green).

**Sources:** Every row needs **≥1** evidence link from: verifier name + script path, CI log, issue/PR URL, doc path + anchor, adoption matrix row id. Mark `source_tier`: **`primary`** | **`mixed`** | **`secondary`** (secondary allowed only with a **promotion hypothesis**).

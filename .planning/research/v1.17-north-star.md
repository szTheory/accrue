# v1.17 — North star and stop rules (FRG-02)

**Status:** Canonical for **Friction-led developer readiness** (Phases **62–65**).  
**Audience:** Maintainers and downstream planning agents. **Integrator-facing** truth remains `accrue/guides/`, host README, and Hex — this file is **milestone doctrine**, not a tutorial.

## North star (one paragraph)

Maximize **marginal developer value**: spend attention where **ranked evidence** shows Phoenix integrators still stall on **integrator / VERIFY / docs**, **billing**, or **admin/operator** — not on broad continuity sweeps after **v1.16**. Each execution phase (**63–65**) is **one sharp bet** on its axis; we **stop** when the next increment removes no meaningful failure mode for the golden path, trust, or merge-blocking contracts.

## Definitions

| Term | Meaning |
|------|---------|
| **Marginal dev value** | Reduction in real integrator time-to-success, verifier honesty, or operator clarity — not line count or subjective polish alone. |
| **Sharp bet** | A phase ships **one axis** (INT-10, BIL-03, or ADM-12) with a clear backlog slice from **FRG-03**, not mixed “misc fixes.” |
| **Diminishing returns** | Further edits would not change **P0** disposition (ship / defer / downgrade) without new evidence. |

## Scope axes

Work is chosen from the **FRG-01** inventory. **INT-10**, **BIL-03**, and **ADM-12** are the **only** execution requirement buckets for **v1.17** P0 closure. **PROC-08** and **FIN-03** remain **explicit non-goals** (see **`.planning/PROJECT.md`**).

## Stop rules (principle + exit test)

| # | Principle | Exit test (binary or queue-based) |
|---|-----------|-----------------------------------|
| S1 | **No broad sweeps without a P0 row** | No multi-file doc rewrite “for consistency” unless a **P0** row in **FRG-01** names the failure and **FRG-03** routes it. |
| S2 | **P0 queue is the intake gate** | Phases **63–65** end with **no open P0** for that axis **or** each remaining row has **maintainer-signed** downgrade / `not_v1.17` in the inventory. |
| S3 | **Merge-blocking contracts stay green** | After any change touching docs or verifiers: **`bash scripts/ci/verify_package_docs.sh`** green; **VERIFY-01** / **host-integration** semantics unchanged unless the same PR updates README + verifier together. |
| S4 | **One sharp bet per phase** | A phase PR set does not expand to a second axis without a **new** requirement row or an explicit milestone amendment. |
| S5 | **Stop at polish without failure modes** | If the only remaining items are **P2** polish with a **documented workaround**, close the phase and record **defer** with **revisit trigger**. |

## Intake and deferrals

- Every **defer** or **not v1.17** row states: **reason**, **revisit condition**, and **future owner** (requirement ID or backlog parking). No tacit drops.

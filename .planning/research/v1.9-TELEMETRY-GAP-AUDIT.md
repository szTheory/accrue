## 2. `Accrue.Telemetry.Metrics.defaults/0` vs ops emits

Default counters (when `:telemetry_metrics` loaded) covered: billing subscription/charge/report_usage, webhook received/dispatched/queue/duration, and ops: DLQ dead/replay/prune, dunning, meter failed, charge_failed, revenue_loss, incomplete_expired.

**Gaps (metrics, pre–v1.9):** No default counters for PDF adapter unavailable, ledger upcast failure, or Connect ops emits.

**Resolution (2026-04-21):** `Accrue.Telemetry.Metrics.defaults/0` now includes counters for all five signals, with low-cardinality tags only where needed (`:type` on PDF unavailable, `:capability` on capability loss). See `accrue/lib/accrue/telemetry/metrics.ex` and `guides/telemetry.md` §Using the default metrics recipe.

## 3. High-signal non-ops `[:accrue, :*]` (alerting usually off)

Worth a **single guide subsection** (not necessarily metrics): webhook orphan reducers, `[:accrue, :webhook, :receive]`, `[:accrue, :webhook, :handler, :exception]`, mailer/PDF/email fallbacks, `[:accrue, :webhooks, :stale_event]`, etc. These are **diagnostic / firehose** — document “subscribe for dashboards, not pages” to match `telemetry.md` philosophy.
